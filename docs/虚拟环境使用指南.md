# 虚拟环境使用指南

本指南介绍如何为 DouK-Downloader 项目创建、激活和管理 Python 虚拟环境。

## 前置要求

- Python 3.12 或更高版本
- 项目要求：`requires-python = ">=3.12"`

检查 Python 版本：

```bash
python3 --version
# 或检查特定版本
python3.12 --version
```

如果未安装 Python 3.12，可通过 Homebrew（macOS）安装：

```bash
brew install python@3.12
```

## 创建虚拟环境

### 方法一：使用 venv（推荐）

```bash
# 进入项目目录
cd TikTokDownloader

# 删除旧的虚拟环境（如果存在）
rm -rf venv

# 使用 Python 3.12 创建虚拟环境
python3.12 -m venv venv
```

### 方法二：使用 uv（更快）

```bash
# 安装 uv（如果未安装）
pip install uv

# 使用 uv 创建虚拟环境并同步依赖
uv sync --no-dev
```

## 激活虚拟环境

### macOS / Linux

```bash
source venv/bin/activate
```

### Windows（PowerShell）

```powershell
.\venv\Scripts\Activate.ps1
```

### Windows（CMD）

```cmd
venv\Scripts\activate
```

激活成功后，终端提示符前会显示 `(venv)`：

```
(venv) user@host:~/projects/TikTokDownloader$
```

## 安装依赖

激活虚拟环境后，安装项目依赖：

```bash
# 升级 pip（可选）
pip install --upgrade pip

# 安装项目依赖
pip install -r requirements.txt

# 或使用国内镜像加速
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
```

## 运行程序

激活虚拟环境并安装依赖后，运行程序：

```bash
python main.py
```

或使用 Python 模块方式运行：

```bash
python -m main
```

## 退出虚拟环境

完成工作后，退出虚拟环境：

```bash
deactivate
```

## 常见问题

### 问题 1：Python 版本不兼容

**症状：** 运行程序时报错或依赖安装失败。

**解决：** 确保虚拟环境使用 Python 3.12+ 创建：

```bash
# 检查虚拟环境的 Python 版本
./venv/bin/python --version

# 如果版本不对，重新创建虚拟环境
rm -rf venv
python3.12 -m venv venv
```

### 问题 2：找不到 python 命令

**症状：** macOS 上系统默认 `python` 命令不存在。

**解决：** 使用 `python3` 或在虚拟环境中使用 `python`：

```bash
# 在虚拟环境中，python 命令可用
source venv/bin/activate
python main.py
```

或在 shell 配置文件中添加别名：

```bash
# 添加到 ~/.zshrc 或 ~/.bashrc
alias python='python3'
alias pip='pip3'
```

### 问题 3：依赖冲突

**症状：** pip 安装依赖时报错。

**解决：** 清理并重建虚拟环境：

```bash
rm -rf venv
python3.12 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

### 问题 4：虚拟环境未激活

**症状：** 运行 `pip install` 时安装到全局环境。

**解决：** 确保激活虚拟环境后再执行命令：

```bash
# 检查当前 Python 路径
which python

# 应显示虚拟环境路径
# 输出：/path/to/TikTokDownloader/venv/bin/python

# 如果显示全局路径，说明虚拟环境未激活
source venv/bin/activate
```

## 快速启动脚本

可以创建启动脚本简化操作：

### macOS / Linux：`run.sh`

```bash
#!/bin/bash
cd "$(dirname "$0")"
source venv/bin/activate
python main.py
deactivate
```

运行：

```bash
chmod +x run.sh
./run.sh
```

### Windows：`run.bat`

```batch
@echo off
cd /d "%~dp0"
call venv\Scripts\activate.bat
python main.py
call venv\Scripts\deactivate.bat
```

运行：

```cmd
run.bat
```

## 附录：虚拟环境目录结构

```

├── venv/                    # 虚拟环境目录
│   ├── bin/                 # macOS/Linux
│   │   ├── activate         # 激活脚本
│   │   ├── python           # Python 解释器
│   │   └ pip              # pip 工具
│   ├── Scripts/             # Windows
│   │   ├── activate.bat
│   │   ├── python.exe
│   │   └ pip.exe
│   ├── lib/                 # 安装的包
│   │   └── python3.12/
│   │       └── site-packages/
├── main.py                  # 主程序入口
├── requirements.txt         # 依赖清单
├── Volume/                  # 数据目录
│   ├── settings.json        # 配置文件
│   └── Download/            # 下载目录
└── ...
```
